Ramas en GIT

git checkout -b rama crea una nueva rama y nos lleva a ella.
git branch muestra las ramas que existen y en cual estamos.

El HEAD apunta al último commit de la rama en la que estamos.
git switch rama nos lleva a otra rama.
git merge rama hace converger otra rama en la que estamos.

git tag "etiqueta" le agrega una etiqueta al HEAD.
Las etiquetas se utilizan en commits importantes y suelen ser numeros de versión como "v0.1.0".
git checkout etiqueta restaura los archivos al estado que tenían en un commit anterior de la rama en la que estamos.
git checkout hash restaura los archivos al estado que tenían en un commit anterior de la rama en la que estamos.
git branch -d rama elimina una rama.
git reset --hard hash nos lleva a un commit anterior de la rama en la que estamos y borra los posteriores al de destino.
Tener mucho cuidado al utilizar los dos comandos anteriores.
git stash crea un commit temporal donde se guardan las modificaciones hechas sobre el HEAD.
git stash list muestra los commits temporales hechos en la rama en la que estamos.
git stash pop restaura las modificaciones guardadas en los commits temporales al volver a la rama donde se los creó.
