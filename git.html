git status muestra en que estado se encuentran los archivos.
git status -s muestra en que estado se encuentran los archivos de forma menos verbosa.
git log --oneline muestra el historial de commits de forma menos verbosa.
git diff muestra los cambios hechos sobre los archivos.
La mayoría de las veces, se muestra que comandos pueden ejecutar a continuación.
Cada commit que se haga va a tener un hash que lo identifica.
En un mismo comando, se puede afectar a más de un archivo si se colocan sus nombres separados por un espacio.
En un mismo comando, se puede afectar a todos los archivos del repositorio utilizando el caracter . como comodín.
